# [보물지도: Family Hub] 요구사항 정의서 (PRD.md)

## 프로젝트 개요

가족 구성원의 일정, 습관(미션), 보상(용돈/쿠폰)을 통합 관리하는 클라우드 기반 웹 앱

## 핵심 타겟 (Persona)

1. **부모 (Admin - dad, mom)**: 전권을 가진 관리자. 일정/미션 등록, 미션 승인 및 이력 관리, 도장/쿠폰 지급, 월말 정산 및 시스템 설정 수행.
2. **자녀 (User - son1, son2)**: 활동 주체. 일정 확인, 미션 수행 보고, 독서 기록/칭찬 기록 및 등록, 본인의 지갑 현황 모니터링.

## 주요 기능 (MVP Scope)

1. **계정별 대시보드**: 로그인 정보에 따라 부모용(관리 UI)과 아이용(개인 현황 UI)으로 자동 전환.
2. **부모**: 일정 등록/관리 -> 이력 테이블에 자동 기록 -> 아이는 등록된 일정 확인.
3. **미션 승인 워크플로우**:
   - 부모: 미션 등록
   - 아이: '미션 클리어' 버튼 클릭 -> 승인 대기 상태 전환.
   - 부모: 요청 알람 확인 -> 승인/거절 버튼으로 처리 -> 이력 테이블 자동 기록.
   - 오늘의 미션 모두 수행 시 도장/쿠폰 선택 지급 및 이력 테이블 자동 기록
4. **독서 및 칭찬 관리**: 아이들이 직접 글을 쓰고 부모가 피드백을 주는 양방향 게시판 형태. 부모도 직접 글을 등록할 수 있음. 이력 테이블 자동 기록.
5. **보상 및 정산 시스템**: 도장(현금 가치)과 게임 쿠폰(시간 가치)을 구분하여 관리. 월말 '정산 완료' 버튼을 통한 장부 마감.
6. **구글 스프레드시트 연동**: 모든 설정(비밀번호, 금액 등)과 이력 데이터는 구글 시트에 실시간 저장.

## 사용자 흐름 (User Flow)

- **로그인**: `streamlit-authenticator`를 통한 세션 유지 (새로고침 시 로그아웃 방지).
- **활동/승인**: 아이가 활동 등록 -> 부모가 대시보드에서 '승인/거절' -> 결과 데이터가 각 항목별 이력 테이블에 즉시 반영. 부모가 오늘의 미션 최종 승인 시 지급되는 도장/쿠폰은 '나의 지갑'에 바로 반영.
- **마감**: 부모가 설정 탭에서 쿠폰/도장 종류 관리 및 월말 최종 정산 처리.
